# Chronovita 项目研发路线图

本文件以申报书《浙江传媒学院大学生创新训练项目申报书》「研究进度安排」为时间锚点，结合工程实施细化为可执行的版本里程碑，并保留「优化构筑记录」追加专区。

> 目标：完成调研收束、工程骨架、四模块最小可用闭环。

| 时间 | 里程碑 | 关键产物 | 预计版本 |
| --- | --- | --- | --- |
| 2026-05 | 实地调研与可行性收束 | 中小学历史教学痛点报告、技术可行性分析 | v1.0.0 |
| 2026-05 | 工程骨架与初始化文档落地 | 前后端骨架、三份核心文档、四个 .pen 设计占位 | v1.0.0 |
| 2026-06 | 看 · 沉浸叙事最小闭环 | Prompt 链、ControlNet 约束器、视频任务队列 | v1.1.0（编排骨架已落地，待接入 Celery + Redis 真异步） |

## 中期规划 · 2026-09 ~ 2026-12

> 目标：交互优化、性能调优、试点学校接入、论文框架成型。

- 2026-09 ~ 2026-10：交互逻辑优化与性能压测
  - 前端 WebGL 渲染管线瓶颈消除，首屏 ≤ 2s
  - 后端推演接口 P95 ≤ 800ms，并发 200 不掉线
  - 智能体语音端到端延迟 ≤ 1.5s
- 2026-09：试点学校（河北定州中学先行）灰度接入与教师培训
- 2026-11 ~ 2026-12：教学测试数据沉淀与论文框架搭建
  - 课堂参与率、知识掌握度、教师满意度三维评估
  - 量化教学效果模型 v1
  - 论文初稿框架：架构创新 · 算法贡献 · 教学实证

## 长期规划 · 2027-01 ~ 2027-04

> 目标：论文成稿、平台精细化、商业化探索、竞赛与结题。

- 2027-01 ~ 2027-02：论文完善与投稿；不同学段教学应用实例补充
- 2027-03 ~ 2027-04：竞赛参赛、结题答辩、平台对外授权探索
  - 商业模式探索：知识付费 · 校本课程开发 · 平台订阅 · 教师授权
  - 内容扩充：从「大禹治水」拓展至「都江堰修建」「商鞅变法」「郑和下西洋」等十大历史推演剧本

## 风险与对冲

| 风险 | 等级 | 对冲策略 |
| --- | --- | --- |
| LLM 历史幻觉导致剧情失真 | 高 | RAG + 规则引擎 + 轻量审核三层；DAG 兜底剧本 |
| 视频生成成本高、出片慢 | 中 | 分级缓存：关键章节预生成、长尾按需生成；Celery 异步队列 |
| 学校落地推进缓慢 | 中 | 先与定州中学小范围深耕，沉淀打磨案例后再扩展 |
| 团队带宽不足 | 中 | 复用既有节点渲染引擎；GenAI 工作流复用既有 ComfyUI 资产 |
| Python 3.14 依赖兼容问题 | 低 | 强制使用 Python 3.12 虚拟环境，规避 pydantic-core/orjson Rust 编译失败 |
