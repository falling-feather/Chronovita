# ADR-0004 问模块双模智者 + 流式对话 + RAG 引证

- 状态：已采纳
- 日期：2026-05-02
- 版本：v1.3.0

## 背景
问模块是史脉「看练问创」中负责对话推理的入口。在没有可调用的本地 LLM 的工程约束下，
仍需让用户感知到「双模制衡 + 引证可复核」这一核心承诺。

## 决策

### 1. 双流派 Persona
- 思辨流派 · 子思（kind=thinker）：开放反问 + 多元解读，0 引证。
- 史实流派 · 仲尼（kind=historian）：以「史载…」开篇，必带引证。
- 二者并置而非切换，让用户感受单一叙事的稀薄。

### 2. Mock LLM 策略
- 思辨派输出由 `_thinker_compose` 模板生成，不依赖外部模型。
- 史实派输出由 `_historian_compose` + `corpus.search_corpus` 关键词命中典籍片段拼接。
- 后续接入真实 LLM 时仅替换 `compose` 与 `search` 两层即可。

### 3. 流式协议
- 后端：`StreamingResponse` + `text/event-stream`，每条 chunk 形如 `data: {json}\n\n`，
  以 `event: end` 收尾。
- 双 persona 通过 `asyncio.Queue` 交叉 yield，前端按 `persona` 字段分流到左右栏。
- 字符块大小 10、间隔 0.05s，模拟打字机节奏。

### 4. 引证结构
- `Citation(source_id, title, excerpt, relevance)`，relevance ∈ [0,1]。
- 关键词命中 → relevance = min(1.0, 0.4 + 0.2·hit)。
- 前端将引证收纳在 Drawer，避免气泡过长。

### 5. 会话存储
- 模块级 `_SESSIONS: dict[str, DialogueSession]`，session_id = `sess_` + uuid[:10]。
- 历史消息在流式结束后落账，保证刷新页面后可回放。

## 后果
- 优点：零外部依赖即可演示「双模 + 引证」体验；引证可点开复核。
- 限制：mock 文本风格固定，需后续接入真实 LLM；语料库目前 7 条，需扩容。
