# ADR-0008 课堂化「老师预设」模块

- 状态：已采纳
- 日期：2026-05-03
- 版本：v1.8.0

## 背景

v1.7.0 之前史脉的 sandbox 始终是「自由探索」模式：学生进入剧本后从默认初始状态开始随意推演。但实际课堂教学中，老师常希望：

1. 让学生在更紧迫/更宽松的初始局面下决策（例如商鞅变法预先把「贵族阻力」拉高，制造抉择压力）
2. 强制学生走完某些必经节点，确保关键史脉不被忽略
3. 限定哪些终局是合格的，避免学生只追求「容易达到的结局」
4. 给出一条推荐路径，事后量化学生与老师设想之间的差距

## 决策

新增 `services/classroom` 模块与配套 router/前端页面，承载「老师预设课堂任务」的全流程。

### 数据模型（`ClassroomTask`）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| task_id | str | `task_<10 hex>` 短哈希，便于学生在 URL 中粘贴 |
| title | str | 任务名 |
| scenario_id | str | 绑定一条 sandbox 剧本 |
| teacher_notes | str | 学生可见的老师备注 |
| preset_state | dict[str,int] | 预调初始状态（按 state_var bits 自动 clamp） |
| must_visit_nodes | list[str] | 必经节点 id |
| accepted_terminals | list[str] | 合格终局 id（空 = 不限） |
| recommended_path | list[str] | 推荐边 id 顺序列表 |

### 验收算法

```
visited = snapshot.history
must_hit  = must ∩ visited
must_miss = must − visited
terminal_accepted = is_terminal AND (accepted_terminals 为空 OR current ∈ accepted_terminals)
ratio = |recommended ∩ traversed_edges| / |recommended|
```

`traversed_edges` 通过遍历 `history[i] → history[i+1]` 反查 `(from,to) → edge_id` 字典得到。

### 与 sandbox 的耦合

- `new_playthrough(scenario_id, preset_state=None)` 增加可选参数；router `POST /playthroughs?task_id=...` 自动从 task 取 preset_state
- 不引入「按 task 演进」的特殊路径；advance 仍走原状态机，不为 task 改写规则
- preset_state 按位宽 `(1<<bits)-1` clamp，避免老师误填超界值导致编码异常

### 前端拆分

- `/classroom`：老师面板，纯 CRUD + 表单
- `/sandbox?task=xxx`：学生页，自动加载任务卡 + 终局后一键验收
- 不在 sandbox 内嵌「创建任务」入口，保持学生侧 UI 简洁

## 替代方案

- **直接在 sandbox 内做老师模式开关**：拒绝。会让学生页混入大量老师才需要的字段，违背单一职责
- **task 注入 advance 路径强制走 must_visit**：拒绝。若强制限制候选边会破坏 DAG 决策训练的本意；改为事后量化
- **recommended_path 用 node 列表**：拒绝。同一对节点之间若有多条边（虽当前剧本暂无）会无法区分；用 edge_id 更稳

## 风险

- 老师误调 preset_state 至「无任何可行边」的死局：当前未做静态可达性检查，记入 backlog
- 学生若不走 `?task=` 链接、自己开新推演验收会失败：UI 已用任务卡 + 验收按钮强提示

## 验证

端到端：创建大禹治水任务（必经 `n_reform/n_field_survey`、合格终局 `n_dynasty`、4 条推荐边）→ 起新推演 → 沿推荐路径走 4 步 → `GET /tasks/{id}/check?playthrough_id=...` 返回：

```json
{
  "is_terminal": true,
  "terminal_accepted": true,
  "recommended_match_ratio": 1.0,
  "must_visit_hit": ["n_reform","n_field_survey"],
  "must_visit_miss": [],
  "summary": "已圆满达成本任务的全部老师预设。"
}
```
