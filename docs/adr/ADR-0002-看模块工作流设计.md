# ADR-0002 看模块工作流设计（v1.1.0）

- 状态：已采纳
- 日期：2026-05-03
- 关联版本：v1.1.0

## 背景

「看 · 沉浸叙事」需要将一段历史文本稳定转化为符合史实形制（服饰、器物、建筑、礼制）的高质量微视频。
直接交给单次 text-to-video 模型存在以下问题：

- 朝代混搭、形制错误（如把唐式建筑放入夏代场景）
- 镜头语言粗糙，缺乏分镜节奏（起 / 承 / 转 / 收）
- 旁白与画面无法对齐，难以服务教学大纲
- 难以审计：单 prompt 黑盒导致教师无法复核与修改

## 决策

采用「Prompt 链 + 分镜编排 + ControlNet 多约束 + 异步任务推进」四段式工作流。

### 1. Prompt 链（services/recall/prompt_chain.py）

将单 prompt 拆解为六段固定角色（system / history_context / scene / character / camera / style），
每段单独模板化，再线性合并为最终 prompt。优点：

- 教师可只编辑「场景」或「人物」段，其他段保持稳定
- 朝代约束（system 段）始终前置，降低混搭概率
- negative prompt 集中维护，覆盖现代物品、文字水印、形制错误等通用噪声

### 2. 分镜编排（services/recall/storyboard.py）

固定四段叙事节奏（起 / 承 / 转 / 收），每段绑定不同镜头语言（中景固定 / 缓慢推镜 / 侧移跟随 / 拉远定格）。
通过启发式扫描历史文本判定是否含人物 / 建筑，决定后续 ControlNet 信号的组合。

### 3. ControlNet 编排（services/recall/controlnet.py）

按场景特征动态组装控制信号：

| 触发条件 | 信号种类 | 默认权重 | 作用 |
| --- | --- | --- | --- |
| 含人物 | pose | 0.85 | 锁定主体姿态 |
| 含建筑 | lineart | 0.70 | 锁定建筑形制 |
| 含建筑 | depth | 0.55 | 稳定透视 |
| 提供参考图 | reference | 0.50 | 风格迁移 |
| 兜底 | scribble | 0.40 | 仅勾稿引导 |

### 4. 异步任务推进（services/recall/pipeline.py）

任务以阶段机模型推进：queued → prompt_chain → storyboard → controlnet → diffusion → animation → tts → compose → done。
v1.1.0 阶段为内存推进占位（每次查询前进一阶），v1.2.0 接入 Celery + Redis 后替换为真实工作流。

## 后果

- 优点：链路可审计、可分段编辑、约束与生成解耦、便于教学场景下的人工校正
- 代价：多模型多阶段编排复杂度高，需要后续接入 Celery + Redis + MinIO
- 风险：ControlNet 模型加载占显存大，v1.2.0 需评估按需加载与卸载策略

## 后续

- v1.2.x：分镜审校面板（教师人工确认）+ 朝代形制规则库（结构化校验）
