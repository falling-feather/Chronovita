# ADR-0001 技术栈选型

- 状态：接受
- 日期：2026-05-03
- 决策人：钮玉彬

## 背景

Chronovita 是面向中小学历史教学的「未来课堂」推演平台，需在团队带宽有限的前提下快速产出可演示 MVP，同时保留中长期向生产级演进的空间。

## 决策

- 前端：React 18 + Vite 5 + TypeScript 5 + Ant Design 5（中式 token 定制）+ Zustand
- 后端：Python 3.12（强制） + FastAPI + Pydantic v2 + SQLAlchemy 2.0
- 数据：PostgreSQL 16 + Redis 7 + Chroma（短期）→ Milvus（中期）
- 部署：Docker Compose（开发） → K8s（远期）

## 后果

正面：
- 类型安全贯穿前后端
- AntD 5 加速 UI 落地，仍可通过 token 实现中式古典调性
- Chroma 零配置上手，迁移到 Milvus 仅需替换 client 适配层

负面：
- Python 3.12 强制要求需提醒新成员（避免 3.14 与 pydantic-core/orjson Rust 编译失败）
- AntD 5 默认 LESS 体积偏大，需要按需引入与 tree-shaking 调优
