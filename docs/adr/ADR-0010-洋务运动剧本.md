# ADR-0010 第四条剧本 · 洋务运动

- 状态：已采纳
- 日期：2026-05-05
- 版本：v2.0.0

## 背景

v1.x 已收敛三条剧本：夏（大禹治水）/ 战国（商鞅变法）/ 北宋（王安石变法）。但样本集中在「先秦—北宋」古典中国，缺少近代史的转型样本，无法演示「器物—制度」二阶矛盾。需引入晚清洋务运动作为第四条剧本，并以此作为 v2.0.0 大版本的标志。

## 决策

### 剧本结构

10 节点 / 12 边 / 5 状态变量（industry / navy / frontier / conservatives / merchant）：

- 起点：`n_zongli`（设总理衙门）
- 强制汇聚：`n_haifang_saifang`（海防塞防之争）——任何主线都需在此抉择
- 互斥分支：`n_xizheng`（左公西征） vs `n_beiyang`（北洋成军），各引一组状态变化
- 二次汇聚：`n_guandu_shangban`（官督商办）
- 终局三选一：
  - `n_jiawu`（甲午之殇）：海军未振或未引商办时触发
  - `n_wuxu_eve`（维新前夜）：器物已成且引商办时触发——隐喻「洋务铺路、戊戌起步」
  - `n_collapse`（顽固反扑）：守旧阻力 ≥4 时初期就被掐死

### 与既有架构的关系

- 不需要修改 engine / router / 前端任何代码，纯数据扩展
- DB 持久化层透明兼容（剧本不入库）
- agent corpus 加 4 条史料增强 RAG（曾/李/左/盛宣怀各一条原典）

## 替代方案

- **加入更多近代节点（戊戌/辛亥/五四）**：拒绝。一条剧本太长会导致 must_visit / recommended_path 难以校准
- **拆成多条小剧本**：拒绝。洋务的核心矛盾就在「器物—制度」张力，需在同一图内呈现互斥与汇聚
- **延后到 v2.1.0**：拒绝。「四条剧本贯通四个史阶」是 v2 的合理标志，不应推迟

## 风险

- 史观争议：海防塞防之争的史界评价并不绝对，剧本将抉择权交给学生而非给定唯一「正确」路径
- 节点数 10 比之前剧本（8/9/11）居中，分支组合 ≈ 12 边在记忆化 DP 下毫秒返回，无性能压力
