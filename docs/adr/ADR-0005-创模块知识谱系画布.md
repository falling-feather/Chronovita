# ADR-0005 创模块知识谱系画布 + 自动布局 + 三件套导出

- 状态：已采纳
- 日期：2026-05-02
- 版本：v1.4.0

## 背景
创模块负责把「看练问」三模块产出的素材沉淀为可复用的知识谱系。需要一个可拖拽编辑、
可自动布局、可导出多格式的画布作为最小可用品。

## 决策

### 1. 数据模型
- `CanvasNodeKind`：event / figure / policy / place / concept 五分类（与中式色映射）。
- `CanvasNode(node_id, kind, label, detail, period, x, y)`：节点位置与画布耦合，避免再做一层映射。
- `CanvasEdge(edge_id, source_id, target_id, label)`：有向 + 文本标签。
- `CanvasBoard`：节点 + 边的纯内存集合，模块级 `_BOARDS: dict[str, CanvasBoard]`。

### 2. CRUD 端点
- 谱系：`GET/POST /boards`、`GET/DELETE /boards/{id}`
- 节点：`POST /boards/{id}/nodes`（upsert）、`DELETE /boards/{id}/nodes/{node_id}`
- 边：`POST /boards/{id}/edges`（upsert）、`DELETE /boards/{id}/edges/{edge_id}`
- 布局：`POST /boards/{id}/layout`（algorithm = layered / radial）
- 导出：`GET /boards/{id}/export?format=json|markdown|mermaid`，`Content-Disposition: attachment` 触发浏览器下载

### 3. 自动布局
- `layered`：Kahn-style BFS 求 layer，按 layer × col 网格布点（x = layer · 240, y = (i - (cnt-1)/2) · 120）。
- `radial`：取度数最大的节点为中心，其余按等角分布在半径 260 的圆上。

### 4. 导出三件套
- `json`：`board.model_dump(mode="json")`，可作为编辑器中转格式。
- `markdown`：按节点类型分组列出 + 「关系」段落呈现边，便于嵌入笔记。
- `mermaid`：`graph LR` + `n_id["label"] -->|label| n_id`，便于嵌入文档站点直接渲染。

### 5. 前端交互
- `reactflow` 复用问/练模块已有依赖，连线 / 拖动 / 删除全部内置。
- 节点单击 → Drawer 编辑；连线点拖到目标节点 → onConnect → POST 边；双击边 → 删除；选中节点按 Delete → 删除。
- 顶部 toolbar 集成新增节点 / 两种布局 / 三种导出。
- 种子谱系「大禹治水 · 夏代奠基」首发 6 节点 6 边，与练模块剧本互文。

## 后果
- 优点：闭环 CRUD + 布局 + 导出，零额外依赖；与四模块色板统一。
- 限制：内存存储，重启即丢；暂未做协作编辑、撤销栈、分组容器。
