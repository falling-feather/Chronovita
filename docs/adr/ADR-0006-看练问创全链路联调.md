# ADR-0006 看练问创全链路联调 + sessionStorage Bridge

- 状态：已采纳
- 日期：2026-05-03
- 版本：v1.5.0

## 背景
v1.0–v1.4 各模块独立可用，但学习者无法把分镜 → 推演 → 论辩 → 谱系串成一气。
为不引入额外状态库或后端长会话，本期需要一个轻量、零依赖的跨页面数据桥。

## 决策

### 1. 桥模块：`apps/web/src/bridge.ts`
- 三段载荷类型：`RecallToSandboxPayload`、`SandboxToAgentPayload`、`AgentToCanvasPayload`
- 每段一对 `set / take` API；`take` 读取后立即从 `sessionStorage` 中清除，避免下次进入页面时重复触发
- 选用 `sessionStorage` 而非 `localStorage`：标签页关闭即清，符合「一次课堂、一段链路」的语义；亦避免需要登录态的复杂度

### 2. 链路接入

| 起点 | 触发 | 落点 | 接收行为 |
| --- | --- | --- | --- |
| RecallPage 分镜生成后 | 「送入练模块 →」按钮 | SandboxPage | 顶部 alert 显示来源 + 关键词；终局沉淀时一并携带 |
| SandboxPage 抵达终局 | 「送入问模块 →」按钮 | AgentPage | 自动设置 topic、预填问题（终局节点叙述）、显示来源关键词 |
| AgentPage 双派输出后 | 「沉淀为创模块谱系 →」按钮 | CanvasPage | 自动 POST 创建 board，再依次 upsert 议题/思辨派/史实派/引证节点，并按 `辩 / 证 / 引` 标签连边，最后调用 layered 自动布局 |

### 3. 首页引导
HomePage 在四宫格下方追加 `Steps` 卡片，明示四步完整链路顺序与按钮名称。

### 4. 边界设计
- 桥接消息只走前端，不污染后端 API 表面
- 引证去重按 `source_id`，最多保留 6 条避免画布过密
- 沉淀创建谱系时所用 API 不依赖任何新端点，复用 v1.4.0 既有 CRUD

## 后果
- 优点：四模块从「独立工具」升级为「一段完整课堂叙事」；零依赖、零接口改动
- 限制：sessionStorage 单标签页生命周期；多次往返会重复创建 board（依赖用户手动删除旧板）

## 验证
- 浏览器实测从首页 → 看（生成大禹治水分镜）→ 练（推演至「开夏之基」终局）→ 问（双派评议 + 3 条典籍引证）→ 创（自动生成 6 节点 5 边「论辩沉淀 · 大禹治水的功过」谱系），全程一键串联
